# Weekly Watch Rewards Design

## Goal

Add a weekly reward system that gives users visible recognition for watching videos. The system should feel rewarding without turning the app into a heavy game system.

The first version focuses on:

- Weekly settled leaderboard
- All-time leaderboard
- Weekly titles
- CSS avatar frames
- Ranking announcement modal
- Admin control for owner participation

## Current Context

The project already has play record data and a play stats feature.

Existing play stats calculate `watchSeconds` from saved play records. The reward system should reuse that watch-duration logic so the ranking numbers match the existing play stats page.

Existing relevant areas:

- `src/lib/play-stats.ts`
- `src/app/play-stats/page.tsx`
- `src/app/admin/user-activity/page.tsx`
- `src/components/UserMenu.tsx`
- `src/lib/admin.types.ts`
- storage adapters under `src/lib/*db.ts`

## Ranking Modes

### Last Week Leaderboard

This is the default leaderboard tab.

- It is generated by a weekly settlement job.
- The settlement runs every Monday.
- It calculates the previous natural week.
- Results stay fixed after settlement.
- Rewards are based on this settled leaderboard.
- Rewards are valid for 7 days.
- The leaderboard shows all eligible users.
- Users with less than 1 hour show `未达标`.

### All-Time Leaderboard

This is a secondary leaderboard tab.

- It is calculated from all saved play records.
- It updates in real time.
- It does not grant rewards.
- It does not show the reward preview block.
- It can still show a user's currently active weekly title/frame if one exists.

## Eligibility

All normal users participate.

Admins participate as ordinary users in the ranking scope they can appear in.

The owner has an admin setting:

- Label: `站长参与排行榜`
- Recommended default: disabled
- When disabled, the owner is excluded from weekly settlement and all-time ranking.
- When enabled, the owner participates in rankings and reward settlement.

This prevents owner testing/playback data from dominating personal-site leaderboards.

## Weekly Reward Tiers

Rewards are based on settled watch time for the previous week.

| Watch Time | Title |
| --- | --- |
| 1 hour | 本周观影者 |
| 3 hours | 本周影迷 |
| 7 hours | 本周追剧达人 |
| 14 hours | 本周放映王 |

Rewards include:

- Title
- Avatar frame

Medals are not part of the first version.

## Avatar Frame Design

Avatar frames are CSS-only. No image upload, R2, or file storage is needed.

Frames wrap the existing avatar area. If the user has no custom avatar, the frame wraps the username initial avatar.

Two sizes are needed:

- `normal`: user menu and user activity detail
- `compact`: leaderboard cards

### Frame Styles

| Tier | Visual |
| --- | --- |
| 本周观影者 | Thin green ring with a small light dot |
| 本周影迷 | Blue segmented tech ring with a subtle inner line |
| 本周追剧达人 | Purple/pink gradient ring with a light orbit and star |
| 本周放映王 | Gold metallic ring with crown and halo |

The compact gold frame uses a smaller halo than the normal frame so it works inside leaderboard rows.

## Rank Awards

Top 3 rank awards are separate from watch-time frames.

Top 3 visual styling uses low-saturation medal styling:

| Rank | Rank Title | Card Style |
| --- | --- | --- |
| 1 | 周榜冠军 | Gold medal rank block with subtle gold card accent |
| 2 | 周榜亚军 | Silver/platinum rank block with subtle silver card accent |
| 3 | 周榜季军 | Bronze rank block with subtle bronze card accent |

The card style communicates rank. The avatar frame communicates watch-time tier.

Example:

- A user ranked first with 7 hours gets:
  - Rank title: `周榜冠军`
  - Watch title/frame: `本周追剧达人`
  - Card style: gold medal rank block
  - Avatar frame: purple tier frame

## Leaderboard Page

Create a user-visible leaderboard page.

All logged-in users can open it.

Tabs:

- `上周榜`
- `全部榜`

Default tab: `上周榜`

Pagination:

- 10 users per page

### Last Week Tab

Each row displays:

- Rank
- Username
- Username initial avatar with active weekly avatar frame
- Rank title when top 3
- Weekly reward title or `未达标`
- Watch duration, placed on the right side on desktop

The bottom of the last-week tab displays reward preview:

- 1 hour: 本周观影者
- 3 hours: 本周影迷
- 7 hours: 本周追剧达人
- 14 hours: 本周放映王
- Text: `奖励有效期 7 天`

When no settlement exists:

- Show an empty state: `暂无上周榜单，等待下次周一结算`

### All-Time Tab

Each row displays:

- Rank
- Username
- Username initial avatar with current active weekly avatar frame when available
- Current weekly title when available
- Total watch duration

This tab does not display reward preview.

## Play Stats Page Change

Remove the existing `用户排行` block from the play stats page.

The play stats page should focus on:

- Overall stats
- Top watched titles
- Recent watch records

User ranking belongs on the new leaderboard page.

## User Menu Display

The user menu should show the current active weekly reward.

Display:

- Avatar with active weekly avatar frame
- Current weekly title

No explanatory text such as `来自上周排行榜`.

If the user has no active reward:

- Show the normal avatar
- Do not show a title badge

## User Activity Admin Display

The admin user activity page should show each user's current active weekly reward.

Recommended display:

- Overview rows: small current title when present
- Detail panel: normal-size avatar frame and title

Admin visibility follows the existing user activity permission model.

## Weekly Settlement Notification

After weekly settlement, users with watch records get a one-time notification modal on next site open.

The modal is generated from the settled result.

It displays:

- Last week watch duration
- Last week rank
- Weekly reward title, or `暂未获得本周奖励`
- Top 3 rank title when present
- Text that rewards were automatically equipped
- Reward validity of 7 days

Example achieved text:

`上周你观看了 7 小时 12 分钟，排名第 1，获得「本周追剧达人」和「周榜冠军」。奖励已自动穿戴，有效期 7 天。`

Example under-threshold text:

`上周你观看了 32 分钟，排名第 8，暂未获得本周奖励。`

Notification rules:

- Generate only for users with watch records in the settled week.
- Do not generate for 0-hour users.
- Users under 1 hour still receive the modal if they watched something.
- Modal is shown once per user.
- Read state is stored server-side.
- The notification expires after 7 days.
- If a user returns after the 7-day window, the old settlement notification is not shown.

## Data Model

Add persistent weekly settlement records.

Recommended shape:

```ts
interface WeeklyWatchSettlement {
  weekStart: number;
  weekEnd: number;
  settledAt: number;
  rewardExpiresAt: number;
  ownerIncluded: boolean;
  rows: WeeklyWatchSettlementRow[];
}

interface WeeklyWatchSettlementRow {
  username: string;
  rank: number;
  watchSeconds: number;
  rewardTier: 0 | 1 | 2 | 3 | 4;
  rewardTitle: string | null;
  rankTitle: '周榜冠军' | '周榜亚军' | '周榜季军' | null;
  notificationReadAt?: number;
}
```

Storage can initially keep one current settlement plus historical settlements if the existing storage layer makes that straightforward.

Minimum first version storage:

- Latest weekly settlement
- Per-user read state within the settlement row

## Backend APIs

Recommended APIs:

- `GET /api/watch-leaderboard?type=weekly&page=1&limit=10`
- `GET /api/watch-leaderboard?type=all-time&page=1&limit=10`
- `GET /api/watch-rewards/current`
- `GET /api/watch-rewards/notification`
- `POST /api/watch-rewards/notification/read`

Admin/site config should include:

- `LeaderboardOwnerParticipates?: boolean`

Weekly settlement can run through the existing cron route style. It should be idempotent for the same week.

## Permissions

Leaderboard page is visible to all logged-in users.

The all-time leaderboard and weekly leaderboard show usernames and watch durations. This is intentional for the reward feature.

Admin user activity remains permission scoped:

- Owner can see all users.
- Admin can see ordinary users and self.
- User cannot access admin user activity.

## Error Handling

Weekly settlement:

- If storage is localStorage mode, disable the leaderboard reward system with a clear API error.
- If no play records exist for the week, generate an empty settlement.
- Running settlement twice for the same week should return the existing settlement or replace it atomically with the same computed result.

Leaderboard page:

- Empty weekly settlement shows the waiting message.
- Empty all-time leaderboard shows `暂无排行数据`.

Notification:

- Expired notification returns no modal.
- Read notification returns no modal.

## Implementation Notes

Start with fixed built-in reward tiers. Admin-editable reward tiers can be added later after the MVP proves useful.

Keep the avatar frame CSS in a reusable component so the same visual is used in the leaderboard, user menu, and user activity admin page.

The leaderboard card's medal styling should remain visually separate from avatar frames.
